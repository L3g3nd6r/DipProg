# DipProg — сборка ПК и ИИ-подбор

Android-приложение + Node.js API. Каталог комплектующих, сборки, корзина, чат с подбором по запросу.

---

## Перенос проекта на другой ПК

Чтобы всё работало так же, как на текущем компьютере, на новом ПК сделайте по шагам:

### Подготовка (один раз на новом ПК)

1. **Установить необходимое ПО**
   - [Android Studio](https://developer.android.com/studio) (для сборки приложения)
   - [Node.js](https://nodejs.org/) LTS (для backend)
   - [PostgreSQL](https://www.postgresql.org/download/) (СУБД для backend)
   - (По желанию) [Ollama](https://ollama.com) — для ИИ-подбора; без неё работает умный fallback по каталогу

2. **Скопировать или клонировать проект**
   - Перенести папку проекта (например с флешки/облака) или:  
     `git clone <URL репозитория> DipProg`  
     и открыть папку в Android Studio / в проводнике.

### Backend (сервер)

3. **Создать файл `backend/.env`**  
   В папке **backend** скопировать `.env.example` в `.env` и подставить свои данные:
   ```env
   PORT=3000
   DATABASE_URL=postgresql://postgres:ВАШ_ПАРОЛЬ@localhost:5432/dipproj
   JWT_SECRET=любая-длинная-секретная-строка
   ```
   Если PostgreSQL на этом ПК с другими логином/паролем/портом — измените `DATABASE_URL` соответственно.

4. **Создать базу и таблицы**
   ```bash
   cd backend
   npm install
   npm run init-db
   npm run init-components
   npm run seed
   ```

5. **Запустить сервер**
   ```bash
   npm start
   ```
   Сервер должен слушать порт 3000.

### Приложение (Android)

6. **Создать файл `local.properties` в корне проекта**  
   Рядом с папками **app** и **backend** создайте файл **local.properties** и укажите адрес API:
   - Эмулятор: можно не создавать (по умолчанию будет `http://10.0.2.2:3000`).
   - Реальное устройство в той же Wi‑Fi, что и ПК:
     ```properties
     api.base.url=http://IP_ЭТОГО_ПК:3000
     ```
     IP ПК: в терминале `ipconfig` (Windows) или `ifconfig`/`ip addr` (Linux/macOS).

7. **Открыть проект в Android Studio и собрать**
   - File → Open → выбрать папку проекта.
   - Дождаться синхронизации Gradle.
   - Build → Rebuild Project.
   - Run (эмулятор или подключённый телефон).

### Что не попадает в Git (нужно настроить вручную на каждом ПК)

| Файл | Назначение |
|------|------------|
| **local.properties** | URL сервера для приложения (`api.base.url`) |
| **backend/.env** | Подключение к БД (`DATABASE_URL`), `JWT_SECRET`, при необходимости `OLLAMA_BASE_URL` |

Имеет смысл заранее записать где-то (в заметках/документе) свои значения `DATABASE_URL`, `JWT_SECRET` и пример `api.base.url`, чтобы на новом ПК быстро подставить их в `.env` и `local.properties`.

## Запуск с реального телефона

Чтобы пользоваться приложением с **реального устройства** (не эмулятора), телефон и компьютер должны быть в одной Wi‑Fi сети.

### 1. Узнайте IP вашего ПК

**Windows (PowerShell или cmd):**
```bash
ipconfig
```
Найдите адрес вида `192.168.1.100` или `192.168.0.10` (раздел «Адаптер беспроводной локальной сети» или «Ethernet»).

### 2. Укажите этот адрес в приложении

В **корне проекта** (рядом с папкой `app`) создайте или отредактируйте файл **`local.properties`** и добавьте строку (подставьте свой IP и порт 3000):

```properties
api.base.url=http://192.168.1.100:3000
```

Файл `local.properties` обычно не попадает в Git — так каждый может указать свой адрес.

Если не создавать `local.properties`, приложение по умолчанию использует `http://10.0.2.2:3000` (только для эмулятора).

### 3. Запустите сервер на ПК

На компьютере:
```bash
cd backend
npm install
npm run init-db
npm run init-components
npm run seed
npm start
```

Сервер должен слушать на `0.0.0.0` или на вашем IP (по умолчанию Express слушает все интерфейсы на порту 3000).

### 4. Соберите и установите приложение на телефон

После изменения `local.properties` пересоберите проект (Build → Rebuild Project) и установите APK на устройство. Приложение будет обращаться к `http://ВАШ_IP:3000`.

## Использование с телефона вне домашней Wi‑Fi (бесплатно)

Если нужно пользоваться приложением **с телефона в другой сети** (мобильный интернет, чужой Wi‑Fi и т.д.), есть два бесплатных варианта.

### Вариант 1: Туннель до вашего ПК (сервер остаётся дома)

Сервер работает на вашем компьютере, а в интернет его «пробрасывает» туннель. Телефон подключается по выданному туннелем URL. **Плюсы:** не нужно переносить код и БД в облако. **Минус:** ПК должен быть включён и на нём должен быть запущен backend.

#### 1.1. ngrok (проще всего)

1. Зарегистрируйтесь на [ngrok.com](https://ngrok.com) (бесплатный аккаунт).
2. Установите ngrok и привяжите токен: [документация](https://ngrok.com/download).
3. Запустите backend на ПК (как в разделе «Запуск с реального телефона»), например на порту **3000**.
4. В отдельном терминале выполните:
   ```bash
   ngrok http 3000
   ```
5. В консоли появится строка вида `Forwarding  https://xxxx-xx-xx-xx-xx.ngrok-free.app -> http://localhost:3000`.
6. В **local.properties** укажите этот **https**-адрес (без слэша в конце):
   ```properties
   api.base.url=https://xxxx-xx-xx-xx-xx.ngrok-free.app
   ```
7. Пересоберите приложение и установите на телефон. Телефон может быть в любой сети.

На бесплатном плане ngrok при каждом запуске даёт новый URL. Если перезапустили `ngrok http 3000` — скопируйте новый URL в `local.properties` и пересоберите приложение **или** откройте в приложении **Настройки → Адрес сервера API**, вставьте `https://…` и нажмите «Сохранить» (без пересборки APK).

#### 1.2. Cloudflare Tunnel (постоянный поддомен, без регистрации в ngrok)

1. Установите [cloudflared](https://developers.cloudflare.com/cloudflare-one/connections/connect-apps/install-and-setup/installation/) (Cloudflare Tunnel).
2. Запустите backend на ПК на порту **3000**.
3. В терминале:
   ```bash
   cloudflared tunnel --url http://localhost:3000
   ```
4. В выводе будет строка вида `https://xxxx-xx-xx.trycloudflare.com` — это ваш публичный адрес.
5. В **local.properties** укажите:
   ```properties
   api.base.url=https://xxxx-xx-xx.trycloudflare.com
   ```
6. Пересоберите приложение.

При каждом новом запуске `cloudflared` адрес может меняться; для постоянного адреса нужна бесплатная регистрация в Cloudflare и настройка своего tunnel.

### Вариант 2: Деплой backend в облако (без ПК)

Backend и база данных живут в облаке. Приложение работает с любого места, ПК можно выключить.

Ниже — **пошаговый сценарий: Neon (PostgreSQL) + Render (Node.js)**. Аналогично можно взять Supabase вместо Neon и Railway / Fly.io вместо Render: логика та же (`DATABASE_URL`, `JWT_SECRET`, публичный **https**-URL сервиса).

#### A. PostgreSQL в облаке (Neon)

1. Зарегистрируйтесь на [neon.tech](https://neon.tech), создайте проект и базу данных.
2. Скопируйте **строку подключения** (`postgresql://…`). Для облачных БД к ней часто добавляют **`?sslmode=require`** (если Neon не дописал параметры сам — см. подсказки в панели).
3. Убедитесь, что доступ с интернета разрешён (у Neon обычно по умолчанию).

#### B. Один раз заполнить БД с вашего компьютера

Удобнее выполнить скрипты **локально** с тем же `DATABASE_URL`, чем настраивать одноразовые команды только на хостинге.

1. В папке `backend`: `npm install`.
2. В PowerShell подставьте свою строку Neon:

   ```powershell
   cd backend
   $env:DATABASE_URL="postgresql://USER:PASSWORD@HOST/DB?sslmode=require"
   npm run init-db
   npm run init-components
   npm run seed
   npm run migrate
   ```

   Порядок: схема и пользователи → каталог/сборки/корзина → сиды → оставшиеся миграции.

3. При ошибках SSL проверьте строку подключения в панели Neon.

#### C. Репозиторий (GitHub)

Залейте проект в Git. Если в репозитории есть и `app/`, и `backend/`, на Render в настройках сервиса укажите **Root Directory** = `backend`.

#### D. Web Service на Render

1. [render.com](https://render.com) → **New** → **Web Service** → выберите репозиторий.
2. **Root Directory**: `backend` (для монорепозитория).
3. **Build Command**: `npm install`  
4. **Start Command**: `npm start`
5. **Environment**:
   - `DATABASE_URL` — та же строка, что в шаге B.
   - `JWT_SECRET` — длинная случайная строка (не используйте значение по умолчанию из кода).
   - **`PORT` не задавайте** — Render подставит порт сам; `server.js` уже использует `process.env.PORT`.

6. После деплоя откройте URL вида `https://имя-сервиса.onrender.com` — по нему должен отвечать API.

**Бесплатный tier Render:** сервис засыпает без запросов; первый запуск после паузы может занять **около минуты**.

#### E. Android-приложение

В **`local.properties`**:

```properties
api.base.url=https://имя-сервиса.onrender.com
```

без слэша в конце. Пересоберите APK **или** укажите тот же URL в приложении: **Настройки → Адрес сервера API**.

#### ИИ (Ollama и облако)

Ollama на бесплатном VPS обычно не ставят. Подбор сборок будет через **встроенный fallback по каталогу**. Облачный ИИ по API можно включить позже переменными из `backend/.env.example` (`OPENAI_API_KEY` и при необходимости `OPENAI_BASE_URL`, `OPENAI_MODEL`) в настройках Render. Для **локальной Ollama** проще **Вариант 1** (туннель к ПК).
