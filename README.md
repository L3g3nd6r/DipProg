# PC Forge — сборка ПК и ИИ-подбор

Android-приложение + Node.js API. Каталог комплектующих, сборки, корзина, чат с подбором по запросу.

---

## Перенос проекта на другой ПК

Чтобы всё работало так же, как на текущем компьютере, на новом ПК сделайте по шагам:

### Подготовка (один раз на новом ПК)

1. **Установить необходимое ПО**
   - [Android Studio](https://developer.android.com/studio) (для сборки приложения)
   - [Node.js](https://nodejs.org/) LTS (для backend)
   - [PostgreSQL](https://www.postgresql.org/download/) (СУБД для backend)
   - (По желанию) [Ollama](https://ollama.com) — для ИИ-подбора; без неё работает умный fallback по каталогу

2. **Скопировать или клонировать проект**
   - Перенести папку проекта (например с флешки/облака) или:  
    `git clone <URL репозитория> PC-Forge`  
     и открыть папку в Android Studio / в проводнике.

### Backend (сервер)

3. **Создать файл `backend/.env`**  
   В папке **backend** скопировать `.env.example` в `.env` и подставить свои данные:
   ```env
   PORT=3000
   DATABASE_URL=postgresql://postgres:ВАШ_ПАРОЛЬ@localhost:5432/dipproj
   JWT_SECRET=любая-длинная-секретная-строка
   ```
   Если PostgreSQL на этом ПК с другими логином/паролем/портом — измените `DATABASE_URL` соответственно.

4. **Создать базу и таблицы**
   ```bash
   cd backend
   npm install
   npm run init-db
   npm run init-components
   npm run seed
   ```

5. **Запустить сервер**
   ```bash
   npm start
   ```
   Сервер должен слушать порт 3000.

### Приложение (Android)

6. **Создать файл `local.properties` в корне проекта**  
   Рядом с папками **app** и **backend** создайте файл **local.properties** и укажите адрес API:
   - Эмулятор: можно не создавать (по умолчанию будет `http://10.0.2.2:3000`).
   - Реальное устройство в той же Wi‑Fi, что и ПК:
     ```properties
     api.base.url=http://IP_ЭТОГО_ПК:3000
     ```
     IP ПК: в терминале `ipconfig` (Windows) или `ifconfig`/`ip addr` (Linux/macOS).

7. **Открыть проект в Android Studio и собрать**
   - File → Open → выбрать папку проекта.
   - Дождаться синхронизации Gradle.
   - Build → Rebuild Project.
   - Run (эмулятор или подключённый телефон).

### Что не попадает в Git (нужно настроить вручную на каждом ПК)

| Файл | Назначение |
|------|------------|
| **local.properties** | URL сервера для приложения (`api.base.url`) |
| **backend/.env** | Подключение к БД (`DATABASE_URL`), `JWT_SECRET`, при необходимости `OLLAMA_BASE_URL` |

Имеет смысл заранее записать где-то (в заметках/документе) свои значения `DATABASE_URL`, `JWT_SECRET` и пример `api.base.url`, чтобы на новом ПК быстро подставить их в `.env` и `local.properties`.

## Запуск с реального телефона

Чтобы пользоваться приложением с **реального устройства** (не эмулятора), телефон и компьютер должны быть в одной Wi‑Fi сети.

### 1. Узнайте IP вашего ПК

**Windows (PowerShell или cmd):**
```bash
ipconfig
```
Найдите адрес вида `192.168.1.100` или `192.168.0.10` (раздел «Адаптер беспроводной локальной сети» или «Ethernet»).

### 2. Укажите этот адрес в приложении

В **корне проекта** (рядом с папкой `app`) создайте или отредактируйте файл **`local.properties`** и добавьте строку (подставьте свой IP и порт 3000):

```properties
api.base.url=http://192.168.1.100:3000
```

Файл `local.properties` обычно не попадает в Git — так каждый может указать свой адрес.

Если не создавать `local.properties`, приложение по умолчанию использует `http://10.0.2.2:3000` (только для эмулятора).

### 3. Запустите сервер на ПК

На компьютере:
```bash
cd backend
npm install
npm run init-db
npm run init-components
npm run seed
npm start
```

Сервер должен слушать на `0.0.0.0` или на вашем IP (по умолчанию Express слушает все интерфейсы на порту 3000).

### 4. Соберите и установите приложение на телефон

После изменения `local.properties` пересоберите проект (Build → Rebuild Project) и установите APK на устройство. Приложение будет обращаться к `http://ВАШ_IP:3000`.
